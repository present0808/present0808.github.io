# Dynamic Programming
Problem Solving을 배우면서 제일 처음 배우는 알고리즘이 DP이다. 그런데 너네도 깨달았겠지만 DP는 난이도의 상한이 없다. 얼마든지 어렵게 만들 수 있다는 소리다. 인터넷에 검색하면 뛰어난 사람들이 만든 DP문제들을 모아 놓은 여러가지 문제집들이 있지만, 선생님이 1년간 고심해서 만든 **중등부 정올을 위한** 문제 리스트는 다음과 같다. 이 난이도보다 어려운 문제가 나오면 그건 3~4번일 것이다.

모든 문제에서 Top-down과 Bottom-up 구현을 해보자. 나도 Top-down을 더 선호하지만 둘 다 쓸 줄 알아야 한다.   
Top-down으로 짰다가 stack 터지면 어쩌려고?  


모든 문제들을 다 숙지하는 것을 추천한다.

int cache[SOME][SIZE];

int dp(int x, int y){
  시작할 때 바닥조건 처리

  int ret;
  입력이 x, y일 때 ret 계산
  return cache[x][y] = ret;
}

# Binary Search, Parametric
100명의 사람에게 이분탐색 코드를 짜보라고 시키면 *완벽하게 오류없이 돌아가도록* 한 번만에 짜는 사람은 10명도 채 되지 않는다. 어려운 이유는 이분탐색을 한 번이라도 짜본 사람이라면 알 것이다.

이게 뭔소리냐? 코드를 **무조건** 외우라는 소리다. 내가 쓰는 템플릿은 다음과 같다.
```
while( low <= high )
{
  mid = (low + high) / 2
  if( function(mid) ) high = mid - 1
  else low = mid + 1
}
```
```low <= high``` , ```high = mid-1```, ```low = mid+1 ```이 핵심이다.
이렇게 하면 while문이 종료되었을 때 low와 high가 역전되는데 그 사이가 딱 O와 X의 경계이다. 이중 1개가 답이 된다.
